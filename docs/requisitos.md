# StockGuard PyME - Requisitos del Sistema

## 1. Requisitos funcionales

### Productos

- El sistema debe permitir registrar productos.
- El sistema debe permitir listar productos.
- El sistema debe permitir editar productos.
- El sistema debe permitir desactivar productos.

### Categorías

- El sistema debe permitir registrar categorías de productos.
- El sistema debe permitir asignar una categoría a cada producto.

### Inventario

- El sistema debe permitir registrar entradas de stock.
- El sistema debe permitir registrar salidas de stock.
- El sistema debe permitir consultar el stock actual de un producto.
- El sistema debe guardar un historial de movimientos de inventario.

### Ventas

- El sistema debe permitir registrar ventas.
- El sistema debe permitir agregar uno o más productos a una venta.
- El sistema debe descontar el stock cuando se registre una venta.
- El sistema debe permitir consultar ventas realizadas.

## 2. Requisitos no funcionales

- El sistema debe ser simple de usar.
- El sistema debe estar pensado para negocios pequeños.
- El sistema debe utilizar una base de datos relacional.
- El sistema debe tener una estructura ordenada de carpetas.
- El sistema debe estar documentado en GitHub.
- El sistema debe poder ejecutarse en un entorno local durante el desarrollo.

## 3. Funcionalidades futuras

- Alertas de stock bajo.
- Reportes de ventas.
- Reportes de productos más vendidos.
- Control de caja diaria.
- Gestión de proveedores.
- Lector de código de barras.
- Control de usuarios y roles.
