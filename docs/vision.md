# StockGuard PyME - Visión del Proyecto

## 1. Problema

Muchos negocios pequeños, como almacenes, bazares, minimarkets y tiendas de barrio, todavía administran sus productos, ventas y stock de forma manual o poco organizada. Esto puede funcionar en algunos casos, pero también puede generar errores importantes, como pérdida de información, falta de control del inventario, productos agotados sin aviso, compras innecesarias o desconocimiento de las ventas realizadas.

El problema principal que busca resolver StockGuard PyME es la falta de una herramienta simple, ordenada y accesible que permita a un negocio pequeño controlar sus productos, movimientos de stock y ventas de manera más eficiente.

## 2. Usuarios del sistema

El sistema está pensado para dueños, administradores o trabajadores de negocios pequeños que necesiten llevar un control básico de sus productos, inventario y ventas.

La aplicación deberá tener una interfaz simple y amigable, para que pueda ser utilizada por personas que no necesariamente tengan conocimientos técnicos avanzados.

## 3. Tipo de negocio objetivo

StockGuard PyME está pensado para negocios pequeños como:

* Almacenes de barrio
* Bazares
* Minimarkets
* Tiendas pequeñas
* Librerías pequeñas
* Ferreterías pequeñas
* Locales con venta de productos físicos

## 4. Objetivo principal

El objetivo principal de StockGuard PyME es permitir que un negocio pequeño pueda registrar y controlar sus productos, ventas y movimientos de inventario de forma ordenada.

El sistema busca ayudar a reducir errores comunes en la administración diaria, entregando información básica pero útil sobre el stock disponible, los productos registrados y las ventas realizadas.

## 5. Funcionalidades iniciales

La primera versión debería permitir:

* Registrar productos con nombre, descripción, categoría, precio y código de barras.
* Consultar el listado de productos disponibles.
* Registrar entradas de stock cuando el negocio compre o reciba mercadería.
* Registrar ventas de productos.
* Descontar automáticamente el stock cuando se registre una venta.
* Revisar el stock actual de cada producto.
* Registrar movimientos de inventario para mantener un historial básico.

## 6. Funcionalidades que NO tendrá al inicio

Por ahora el sistema no incluirá:

* Boleta electrónica real.
* Integración con el Servicio de Impuestos Internos.
* Pagos con tarjeta o integración con Transbank.
* Lector físico de código de barras.
* Control avanzado de usuarios y permisos.
* Alertas por fechas de vencimiento.
* Sistema contable avanzado.
* Aplicación móvil.

Este proyecto me sirve para demostrar que puedo diseñar y desarrollar una aplicación informática orientada a resolver un problema real de un negocio pequeño.

Con StockGuard PyME podré demostrar conocimientos en:

* Análisis de requerimientos.
* Diseño de base de datos.
* Programación backend.
* Creación de una API.
